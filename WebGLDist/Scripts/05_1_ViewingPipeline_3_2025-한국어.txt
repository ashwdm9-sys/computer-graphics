컴퓨터 그래픽스에서 카메라는 중요한 역할을 합니다
같은 오브젝트들이 3D 월드에 고정되어 있다 하더라도
그 오브젝트들을 어떻게 바라보는가에 따라
최종 아웃풋 이미지는 달라지게 됩니다
이것은 마치 인물 사진을 찍을 때
카메라의 위치와 방향을 어떻게 하느냐에 따라
같은 인물을 찍더라도 찍힌 사진이 달라지게 되는 것과 같습니다
그림은 Unity 게임 엔진의 game view 와 working view 를 보여주고 있습니다
이 두 가지 view 들에 사용된 카메라들은 서로 다른 것입니다
Working view 에서 흰색으로 표시된 카메라를 어느 위치에 두고
어디를 바라보게 하느냐에 따라
위쪽 game view 에서 실제 게임에 보이는 scene 은 달라지게 됩니다
working view 의 흰색 카메라의 아이콘의 방향은
실제 카메라가 바라보는 방향은 아닙니다
다만 카메라의 아이콘이 그 방향으로 고정되어 있기 때문에
그렇게 표시되는 것입니다
여기서 카메라는 두 개의 구 쪽을 바라보고 있고
그 때문에 위쪽 윈도우의 game view를 볼 수 있게 됩니다
이전에 언급했듯이 OpenGL에서의 카메라는 항상 Origin에 위치하고
-Z 방향을 바라보고 있습니다
이 위치와 방향을 우리는 바꿀 수 없습니다
그러면 원하는 위치와 방향으로 카메라를 옮겨서
원하는 view를 볼 수 있게 하려면 어떻게 해야 할까요?
그것은 카메라의 모션을 시뮬레이션 하는 것입니다
즉, 현재 view 에 보이는 오브젝트를
카메라의 모션의 반대 방향으로 움직임으로써
카메라가 움직이는 것처럼 느끼게 하는 것입니다
이것은 영화 촬영 때 기차 내부 세트를 만들었을 때
창밖에 나무들을 움직이게 하면 기차가 가고 있는 느낌을 주는 것과 같습니다
다시 말하면 카메라를 배치하는 대신에
반대로 오브젝트들을 배치하자는 아이디어입니다
카메라의 space 를 정의하기 위해
카메라의 각종 parameter 들을 world space 상에서 정의합니다
우리가 잘 알고 있는 World Coordinate System은
서로 수직인 X, Y, Z 축을 베이시스로 가지고 있으며, 오른손 좌표계입니다
즉, Z 축의 양의 방향이 스크린으로부터 우리 쪽을 향하고 있습니다
카메라를 정의하기 위해 몇 개의 중요한 파라미터들을 정의합니다
P는 카메라의 위치로 Center OF Projection, 줄여서 COP라 불립니다
A는 카메라가 바라보고 있는 target point 를 나타내는데,
여기서는 At Point라 부르겠습니다
V는 UP vector인데 그림에서 보듯 카메라의 머리 위쪽 방향을 나타내는 벡터입니다
이제 위의 파라미터들을 이용하여 view,
즉 Camera (Eye) coordinate 시스템을 정의해 보겠습니다
이 좌표계의 세 축들 U, V, N 을 구할 것입니다
U는 카메라의 오른쪽 방향의 축벡터인데,
일반적인 좌표계로 보면 X축에 해당합니다
V는 아까 정의한 것과 같이 카메라의 위쪽,
즉 Y쪽 방향을 나타내는 축벡터입니다
마지막으로 N은 카메라가 바라보고 있는 방향의 축을 나타내는데,
일반적인 좌표계의 +Z 축에 해당합니다
N vector 는 P 로부터 A 로 가는 vector가 됩니다
또한 N은 view plane 의 normal vector 와 같습니다
그런데 그림에서 보면 +N 방향이 world space 와 반대이고
+N 방향이 우리 쪽에서 화면의 안쪽 방향을 향하고 있습니다
즉 View coordinate system 은 왼손 좌표계라 할 수 있습니다
이러한 UVN 의 정의를 바탕으로 실제 UVN 을 구하는 방법을
다음 슬라이드에서 살펴보기로 합니다
이제 주어진 카메라 파라미터들 P, A, V로부터
UVN 축들을 계산하는 방법을 살펴보도록 합니다
먼저 up vector V 대신에
parameter 로 virtual up vector V' 이 주어졌다고 가정하겠습니다
v' 은 정확한 V 방향 벡터가 아닌
대략적인 UP 방향을 나타내는 근사치의 벡터입니다
사실 P로부터 A로 가는 방향의 벡터가 N인데
N과 V는 서로 수직이어야 합니다
그러나 이런 벡터를 user 가 직접 계산해서
카메라 파라미터로 입력하기는 쉽지 않습니다
따라서 대충 비슷한 v' 을 주고
정확한 V를 프로그램이 계산하도록 하는 것입니다
많은 경우에 world space 의 Y축 (0, 1, 0) 이 v' 으로 사용됩니다
물론 카메라를 거꾸로 들고 있는 경우라면 -Y축이 더 나을 수도 있겠습니다
N의 계산은 가장 쉽습니다
P로부터 A로 가는 벡터 (A - P) 를 n 이라 하면
n 을 normalize 하면 N 이 됩니다 
U vector 는 V 와 N 에 모두 수직입니다
우리는 n과 v'을 cross product 하여 u 를 구하고
이를 normalize 하여 U 를 구합니다
좀 자세히 보면 이상한 점을 발견할 수 있는데요
UVN 좌표계 이므로 U 를 구하려면
N x V가 아니라 V x N을 해야 하는 게 아닌가 하는 생각이 들 것입니다
그런데 그것은 오른손 좌표계의 경우입니다
앞에서 보았듯이 카메라 스페이스의 UVN은 왼손 좌표계 이기 때문에
V x N 의 반대 방향인 N x V 를 계산하여 U 를 구하였습니다
이제 N과 U가 정해졌으니까 정확한 V를 구할 수 있습니다
u x n을 하여 v를 구하고,
이를 normalize 하면 V 가 됩니다
위와 마찬가지로 오른손 좌표계와는 반대 방향으로
cross product 를 계산하기 때문에
n x u 가 아니라 u x n 을 계산합니다
view space 의 UVN 축이 결정되었으면
World space 에서 view space 로의 좌표 변환이 가능해집니다
이제 우리는 바로 앞 note,
05-1 Viewing Pipeline - 1 에서 언급했던
View matrix 를 결정할 수 있게 되었습니다
먼저 앞에서 설명한 방법으로 UVN이 구해졌습니다
또 COP P의 좌표도 주어졌습니다
이때 P와 UVN의 모든 좌표들은 world space 를 기준으로 표현된 것입니다
또 모든 UVN은 모두 unit vector 들입니다
이전에 공부한 Change OF Frame에 따르면
View Coordinate로부터 World Coordinate로 변환하는
Transformation Tcw는 이와 같이 주어집니다
Tcw = Tp R 인데 
UVN vector들과 P의 coordinate들을 Matrix의 각 칼럼으로 배치하는
일반적인 Rotation Before Translation 형태의 Matrix입니다
우리의 원래 목표는 위와 반대 방향인 World to View Space 였으므로
Twc matrix 는 Tcw 의 inverse 를 이와 같이 구하면 됩니다
rotation matrix 의 inverse 는 그 matrix 의 transpose 와 같고
translation matrix 의 inverse는 vector 방향을 반대로 하면 된다는 것을 기억하면
쉽게 이해할 수 있습니다
앞에서 설명한 과정을 실제 예로 살펴보도록 합니다
계산을 간단히 하기 위해 COP 인 P는 world coordinate로 (0, 0, 5)
즉, +Z 축 상에 있고
카메라는 현재 world space 의 원점을 보고 있다고
즉, A = (0, 0, 0) 이라고 가정하겠습니다
또 up vector는 그냥 Y축과 같다고 가정해보겠습니다
그림을 참고하기 바랍니다
먼저 N = A - P 이므로
N = (0,  0, -1)  즉 world의 minus Z 축이 됩니다
U = N x V' 이므로 
U = (1, 0, 0) 즉 world의 X축과 같습니다
마지막으로 V = U x N 을 계산해보면
역시 처음 가정했듯이 V는 world의 Y축이 됩니다
이제 world space의 한 점 Qw = (3, 3, -1) 을
view coordinate로 바꾸어 보겠습니다
앞에 슬라이드에서 유도한 것처럼 view coordinate Qc 는
Twc matrix와 Qw 의 곱과 같습니다
이 식을 모두 기억할 수 있을 것입니다
여기에 바로 앞장에서 계산한 UVN과 P 좌표 값들을 넣어주면
Qc 는 (3, 3, 6, 1) 이 나옵니다
사실 이 좌표 변화는 계산 없이도 직관적으로 가능합니다
카메라가 World의 +Z 축의 (0, 0, 5) 에 있고 원점을 바라보고 있으므로
Qw의 Z 좌표 -1은 Qc 에서 6이 될 것입니다
나머지 U와 V는 변화없이 3으로 남아 있다는 것도
쉽게 이해할 수 있을 것입니다
이번 예제에서는 COP 의 좌표를 (1, -1, 5) 로 약간 바꾸었습니다
이렇게 하면 view space는 약간 비틀어지고 NUV 축들도 바뀌게 됩니다
계산하는 방법은 이전과 같습니다
